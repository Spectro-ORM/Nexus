# ADR-005: Package Target Layout — Three Targets in One Repository

**Status:** Accepted
**Date:** 2026-03-25
**Owner:** Alembic Labs

---

## Context

Nexus has three layers with distinct dependency profiles:

| Layer | Description | External Deps |
|---|---|---|
| Core | `Connection`, `Plug`, `Body`, pipeline helpers | `swift-http-types` only |
| Router DSL | Result-builder HTTP router | `Nexus` only |
| Hummingbird adapter | Bridges Nexus to Hummingbird's request lifecycle | `Nexus` + `Hummingbird` |

Options considered:

- **Three separate repositories** — maximum decoupling, but imposes a painful release
  coordination burden for a project at this stage. Consumers would need to pin three
  separate packages.
- **One repository, one library target** — all code in a monolith. Consumers of the core
  are forced to pull in Hummingbird even if they use a different server framework.
- **One repository, three library targets** — consumers declare exactly the targets they
  need; Hummingbird is not pulled in unless `NexusHummingbird` is declared as a dependency.

## Decision

The `swift-nexus` repository contains a single `Package.swift` declaring three library
targets:

- **`Nexus`** — pure core. Only external dependency is `swift-http-types`.
- **`NexusRouter`** — result-builder DSL router. Depends on `Nexus` only.
- **`NexusHummingbird`** — Hummingbird server adapter. Depends on `Nexus` + `Hummingbird`.

Source layout:

```
Sources/
  Nexus/
  NexusRouter/
  NexusHummingbird/
Tests/
  NexusTests/
  NexusRouterTests/
```

A `NexusHummingbirdTests` target is deferred until Sprint 2 when the adapter has
testable surface area.

## Consequences

**Positive**
- Consumers who only need the core (`Nexus`) do not pay the cost of fetching or
  compiling Hummingbird.
- A single `Package.swift` means one version tag covers all three targets — no release
  coordination overhead.
- It is straightforward to split a target into its own repository later if the project
  grows large enough to warrant it.

**Negative / Trade-offs**
- `NexusHummingbird` and `NexusRouter` are versioned together with the core. A breaking
  change to the router API requires bumping the major version even if the core is
  unchanged. Accepted as an appropriate trade-off for a 0.x release series.
