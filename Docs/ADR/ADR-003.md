# ADR-003: Plug Is a Function Typealias, Not a Protocol

**Status:** Accepted
**Date:** 2026-03-25
**Owner:** Alembic Labs

---

## Context

The central abstraction in Nexus is the unit of middleware — something that receives a
`Connection` and returns a transformed `Connection`. We need to decide whether this unit
is:

- A **protocol** (e.g. `protocol Plug { func call(_ conn: Connection) async throws -> Connection }`)
- A **function typealias** (e.g. `typealias Plug = @Sendable (Connection) async throws -> Connection`)

Elixir's Plug uses module-based composition (a protocol analog). Phoenix plugs are either
functions or modules — function plugs are idiomatic for simple transformations.

In Swift, the protocol approach introduces:
- Existential overhead (`any Plug`) or generics proliferation (`<P: Plug>`).
- Boilerplate conformance declarations for every plug.
- Complexity in result-builder DSLs that compose heterogeneous plug types.

## Decision

`Plug` is a function typealias:

```swift
public typealias Plug = @Sendable (Connection) async throws -> Connection
```

Any Swift function, closure, or method with this signature is automatically a `Plug` —
no conformance declaration required. Pipeline composition is done with free functions
(`pipe`, `pipeline`) and, in `NexusRouter`, a result builder.

## Consequences

**Positive**
- Zero boilerplate — any matching closure is a plug.
- Trivially composable with higher-order functions.
- `@Sendable` in the typealias enforces concurrency safety at the call site; no
  `Sendable` conformance annotations needed on plug types.
- Closures capture state naturally; no need for stored-property protocol witnesses.

**Negative / Trade-offs**
- Plugs have no identity — you cannot introspect a pipeline to ask "does this pipeline
  contain a `CORSPlug`?". This is an acceptable trade-off; middleware introspection is
  rarely needed and can be addressed via the `assigns` dictionary if required.
- Protocol-oriented middleware frameworks (e.g. Vapor's `Middleware`) cannot be adopted
  directly; an adapter wrapper is required.
