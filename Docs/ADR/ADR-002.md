# ADR-002: Body Is an Enum, Not a Concrete Type

**Status:** Accepted
**Date:** 2026-03-25
**Owner:** Alembic Labs

---

## Context

HTTP bodies have three distinct shapes at the point a plug pipeline handles them:

1. **Absent** — a GET request, a 204 response, etc.
2. **Buffered** — the full body has been read into memory (small payloads, unit tests).
3. **Streaming** — the body arrives or is produced incrementally (large uploads/downloads,
   server-sent events, chunked transfer).

The body representation needs to:
- Be `Sendable` for Swift 6 strict concurrency.
- Make the "absent" case cheap (no allocation).
- Not force all plugs to handle streaming — most plugs only care about buffered bodies.
- Compose naturally with `Connection` as a value type.

Alternatives considered:
- A protocol with conformances (`EmptyBody`, `BufferedBody`, `StreamingBody`) — leads to
  type-erasure overhead and complex generics on `Connection`.
- A single concrete struct with optionals — blurs the distinction between absent and
  buffered; produces confusing APIs.

## Decision

`RequestBody` and `ResponseBody` are each a `public enum` with three cases:

```swift
public enum RequestBody: Sendable {
    case empty
    case buffered(Data)
    case stream(AsyncThrowingStream<Data, any Error>)
}

public enum ResponseBody: Sendable {
    case empty
    case buffered(Data)
    case stream(AsyncThrowingStream<Data, any Error>)
}
```

They are separate types (not a single shared `Body` enum) because request and response
bodies may acquire different helper methods and semantic meanings over time.

## Consequences

**Positive**
- Exhaustive `switch` statements make it impossible to silently ignore the streaming case.
- Value-type composition — `Connection` stays a plain `struct`.
- `Sendable` conformance is synthesized automatically (all cases are `Sendable`).
- `.empty` has zero allocation cost.

**Negative / Trade-offs**
- Callers must `switch` or use pattern matching to access body data — slightly more
  verbose than a property accessor on a concrete type.
- `Data` (Foundation) is the buffer type. If non-Foundation targets are needed in the
  future, `ByteBuffer` or `[UInt8]` may be preferable. This is deferred to a future ADR.
